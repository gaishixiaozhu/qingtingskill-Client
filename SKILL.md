# 蜻蜓志愿数据查询 Skill v3.0

## 功能

蜻蜓生涯志愿填报数据库全功能查询服务。支持全国31省市区。只要是志愿填报相关数据查询，都通过此 Skill 处理。

**v3.0 升级**：客户端不再透传服务端纯文本答案，而是用 `display` 结构化数据渲染精美输出（表格/分类/颜色标记）。

## 核心原则

⚠️ **原封不动转发**：用户问题完整转发给服务器，不修改、不重写、不添加任何省份/科类/分数推断。
⚠️ **不做任何上下文推断**：不从历史对话中补充信息，不根据记忆猜测省份。

## 支持的查询类型

| 查询类型 | 能做什么 | 示例问题 |
|---------|---------|---------|
| **志愿推荐** | 根据分数/位次推荐院校专业，按冲刺/适合/稳妥三层分类 | "山东物理600分推荐计算机" |
| **招生计划** | 查某校某专业在某省的招生人数 | "大连理工在辽宁招多少人" |
| **历史分数** | 查院校专业历年录取分数线 | "华中科技计算机去年多少分" |
| **一分一段表** | 分数↔位次互查 | "浙江600分排多少名" |
| **批次线** | 查各省各批次省控线 | "四川2025年一本线多少" |
| **院校信息** | 查院校代码、层次、所在地等 | "深圳大学是双一流吗" |
| **专业查询** | 查专业介绍、学什么、就业方向 | "数据科学就业方向" |

## 配置


config.py:
```python
SERVER_URL = "https://ocs.qingtingai.pro"
```

## 志愿推荐三必填条件

生成志愿方案必须确认：
- ① **考生省份**（如山东、广东、四川）
- ② **选科**（物理类/历史类，或3+3/文科/理科）
- ③ **分数或位次**

缺少时自动追问补全。

## API Key

联系管理员获取。每个 Key 有独立的日配额和月配额。
