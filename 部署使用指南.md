# Client OpenClaw Skill 部署使用指南

## 一、功能说明

「蜻蜓志愿数据查询」是安装在 Client OpenClaw 上的 Skill，功能：
- 验证 API Key（识别客户身份）
- 接收用户提问
- 转发到 Server OpenClaw
- 原封不动返回答案
- 三必填条件提醒

---

## 二、适用对象

- **已有 OpenClaw** 的客户（企业/个人）
- **第三方系统** 希望通过 OpenClaw 接入志愿查询服务

---

## 三、安装步骤（Client OpenClaw）

### 方式A：手动安装

#### 1. 复制Skill文件

将以下文件复制到 OpenClaw 的 skills 目录：

```
~/.openclaw/workspace/skills/
└── qingting_data_query/
    ├── SKILL.md
    └── qingting_query_skill.py
```

#### 2. 配置API Key

编辑 Skill 配置文件或 OpenClaw 环境变量：

```bash
# 方式1：环境变量
export QINGTING_API_KEY="tk_你的APIKey"
export QINGTING_SERVER_URL="http://服务器IP:5007"

# 方式2：Skill配置界面
# 在OpenClaw管理界面找到该Skill，填写配置
```

#### 3. 重启OpenClaw

```bash
openclaw restart
```

## 四、使用方法

### 基本使用

客户在 OpenClaw 中直接提问：

```
辽宁物理类520分能上什么学校？
北京历史类580分能上哪些大学？
帮我推荐志愿
```

Skill会自动：
1. 检查 API Key 配置
2. 检查三必填条件
3. 转发到 Server
4. 返回答案

### 首次使用提示

如果客户未配置 API Key：
> ⚠️ 请先配置您的API Key才能使用蜻蜓志愿数据查询服务。
> 
> 如需API Key请联系管理员。

### 三必填条件提醒

当客户请求「推荐志愿」但缺少信息时：

**缺少省份：**
> ⚠️ 为了给您提供准确的推荐，请补充以下信息：
> 1. 省份
> 
> 请告诉我您的完整信息，例如：辽宁物理类520分能上什么学校？

**缺少科类：**
> ⚠️ 为了给您提供准确的推荐，请补充以下信息：
> 1. 省份
> 2. 科类（物理类/历史类）
> 
> 请告诉我您的完整信息，例如：辽宁物理类520分能上什么学校？

**缺少分数：**
> ⚠️ 为了给您提供准确的推荐，请补充以下信息：
> 1. 省份
> 2. 科类（物理类/历史类）
> 3. 分数或位次
> 
> 请告诉我您的完整信息，例如：辽宁物理类520分能上什么学校？

## 七、第三方系统接入

### OpenClaw Bot接入

在扣子平台创建Bot，配置HTTP插件：

**插件配置：**
- 请求地址：`http://你的服务器IP:5007/api/v1/chat`
- 请求方法：POST
- 认证方式：Header `X-API-Key`
- 请求体格式：
```json
{
  "user_id": "${schema.user_id}",
  "session_id": "${schema.session_id}",
  "question": "${message.content}"
}
```

### 自有系统接入

参考 Server OpenClaw 的 API 接口文档：
- [Server OpenClaw部署使用指南.md](../Server%20OpenClaw/部署使用指南.md)

**Python示例：**
```python
import requests

api_key = "tk_xxxxx"
server_url = "http://服务器IP:5007"

resp = requests.post(
    f"{server_url}/api/v1/chat",
    json={
        "user_id": "your_user_id",
        "session_id": "your_session_id",
        "question": "辽宁物理类520分能上什么学校"
    },
    headers={"X-API-Key": api_key}
)

result = resp.json()
print(result["answer"])  # 直接打印答案
```

## 八、常见问题

### Q1：提示"请先配置API Key"

**原因：** 未配置 QINGTING_API_KEY 环境变量

**解决：**
```bash
export QINGTING_API_KEY="tk_你的APIKey"
export QINGTING_SERVER_URL="http://服务器IP:5007"
openclaw restart
```

### Q2：提示"无法连接到服务器"

**原因：**
1. Server URL 配置错误
2. 服务器端口未开放
3. 服务器宕机

**检查：**
```bash
# 测试服务器连通性
curl http://服务器IP:5007/api/v1/health

# 检查API Key验证
curl "http://服务器IP:5007/api/v1/key/verify?api_key=tk_xxxxx"
```

### Q3：返回"API Key无效"

**检查：**
1. API Key是否正确粘贴（注意不要有空格）
2. Key是否已过期
3. Key是否被禁用

**联系管理员**确认Key状态

### Q4：返回"日配额已用完"

**原因：** 该API Key当日使用次数已达上限（默认100次/天）

**解决：**
- 等待次日自动恢复
- 联系管理员提高配额

### Q5：答案格式异常

**原因：** Server OpenClaw 返回格式变化

**解决：** 检查 Server 日志，联系技术支持
